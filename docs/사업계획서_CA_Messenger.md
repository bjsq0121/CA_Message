# CA Messenger 사업계획서

## 1. 프로젝트 개요

### 1.1 프로젝트명
**CA Messenger** — 기업 전용 실시간 메신저 플랫폼

### 1.2 프로젝트 배경
- 사내 커뮤니케이션이 카카오톡, 문자, 유선 전화 등 분산된 채널에 의존
- 업무 대화와 개인 대화가 혼재되어 보안 및 관리 이슈 발생
- 기존 ERP(CA_NERP) 사용자 체계와 연동된 통합 메신저 부재
- 외부 메신저(Slack, Teams) 도입 시 라이선스 비용 및 커스터마이징 한계

### 1.3 프로젝트 목표
- ERP 사용자 인증 연동 기반의 **사내 전용 메신저** 구축
- **웹, 데스크톱(Windows/Mac), 모바일(iOS/Android)** 멀티 플랫폼 지원
- 기존 ERP 시스템과의 **MSA(마이크로서비스)** 기반 독립 운영
- 부서/조직 기반 채팅방 자동 관리로 업무 효율 극대화

### 3.3 기술 스택
| 계층 | 기술 | 선정 이유 |
|------|------|----------|
| 프론트엔드 | Vue 3 + TypeScript | 경량, 빠른 개발, 기존 ERP(Vue 2)와 유사 |
| 백엔드 | Node.js + Express | Ably SDK 최적 지원, 비동기 처리 |
| 실시간 통신 | Ably Chat SDK | 글로벌 인프라, 안정성, 확장성 |
| 데이터베이스 | SQLite(개발) → Oracle(운영) | ERP DB 통합 운영 |
| 데스크톱 | Electron | 크로스 플랫폼, 웹 코드 재사용 |
| 모바일 | Capacitor | 네이티브 앱 빌드, 웹 코드 재사용 |
| 인증 | JWT | CA_NERP와 동일 인증 체계 |

### 3.4 보안 체계
- **인증**: CA_NERP JWT 토큰 기반 인증 (동일 Secret Key)
- **퇴사자 차단**: USE_YN = 'Y' 사용자만 접근 허용
- **데이터 주권**: 모든 데이터 사내 IDC Oracle DB 저장
- **통신 암호화**: HTTPS + Ably TLS 암호화
- **세션 관리**: 8시간 토큰 만료, 자동 로그아웃

## 9. 결론

CA Messenger는 기존 ERP 인프라를 최대한 활용하여 **추가 라이선스 비용 없이** 사내 전용 메신저를 구축하는 프로젝트입니다.

**핵심 가치:**
1. **비용 절감** — 외부 서비스 대비 연 ₩14,700,000 절감
2. **보안 강화** — 사내 IDC 데이터 주권 확보
3. **ERP 통합** — 사용자/조직 체계 자동 연동
4. **멀티 플랫폼** — 웹, 데스크톱, 모바일 통합 지원
5. **확장성** — MSA 구조로 독립 운영 및 확장 용이

MVP 개발이 완료된 상태이며, IDC 배포 및 베타 테스트를 거쳐 전사 도입을 목표로 합니다.
